# -IT-Slayers
 	Здравствуйте, в данном файле мы отметим, какие у нас файлы и за что отвечают.
	
В связи с тем, что компетенции команды не позволяют реализовать
динамическое изменение маршрута, путем ввода нужных точек маршрута напрямую в веб-сервисе
и последующего его вывода, было принято решение написать отдельный
алгоритм на C++. Вводя вершины и веса графа в код программы можно на выходе получить оптимальный маршрут из точки А в точку Б.
Также объем работы по составлению графов оказался
трудоёмким и охватить 70% всего города Липецк - невозможно. 
Поэтому было принято решение зацепить 4 квартала города Липецк (это примерно 17% всего города), оставшаяся часть города строилась бы аналогичным образом.
Нулевая отметка (начальная вершина графа) - гараж с спецтехникой (рассчитанный на 102 машины).

Далее приведена информация с алгоритмом составления оптимальных маршрутов. 
weight при запуске программы - вес графа, кратчайшее кол-во переходов между вершинами.
1)Hack.rar (анализ маршрута для 1 группы уборочных машин) (запускать в архиве Hack/Debug/Hack.exe или можно посмотреть сам код в visual studio файл Hack.sln)
2)Hack2.rar (анализ маршрута для 2 группы уборочных машин) (запускать в архиве Hack/Debug/Hack2.exe или можно посмотреть сам код в visual studio файл Hack2.sln)
3)Hack3.rar (анализ маршрута для 3 группы уборочных машин) (запускать в архиве Hack/Debug/Hack3.exe или можно посмотреть сам код в visual studio файл Hack3.sln)
4)"Сайт обновленная версия.rar" (веб-сервис для оптимальной работы с маршрутом спецтехники)(Возможно запустить через Visual Code, нажав по файлу index.html, или нажать на index.html и выбрать просмотр в браузере) (также можно перейти по ссылке DanseMecabre.github.io)
